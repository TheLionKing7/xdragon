# Commit Message Convention

## Context

Ollama's [CONTRIBUTING.md](../../CONTRIBUTING.md) defines a specific commit message format. The title must follow:

```
<package>: <short description>
```

Where `<package>` is the most affected Go package (e.g. `server`, `api`, `convert`, `middleware`). If the change does not affect Go code, use the directory name instead. The short description must start with a lowercase letter and be a continuation of the sentence "This changes Ollama to..."

Bad prefixes like `feat:`, `fix:`, `chore:` are explicitly called out as incorrect examples in CONTRIBUTING.md.

## What to Check

### 1. Commit Title Format

Every commit message title must match the pattern `<package>: <lowercase description>`.

**GOOD:**
```
server: add panic recovery middleware
convert: support openclaw model architecture
app/ui: add creative playground route
```

**BAD:**
```
feat: add more emoji
fix: was not using famous web framework
chore: generify code
Add Creative Playground
fortress: harden system with resilience middleware
```

### 2. Package Name Accuracy

The package prefix should refer to a real Go package or directory in the repo:
- `server`, `api`, `cmd`, `convert`, `middleware`, `openai`, `model`, `runner`, `sample`, `kvcache`, `ml`, `auth`, `envconfig`, `format`, `manifest`, `parser`, `progress`, `template`, `thinking`, `tokenizer`, `tools`, `types`, `version`
- Or directory paths: `app/ui`, `docs`, `scripts`, `integration`, `x/agent`
- Single root files: `CONTRIBUTING`, `README`, `Dockerfile`, `Makefile.sync`

### 3. Description Style

- Must start with a **lowercase** letter
- Should complete the sentence "This changes Ollama to..."
- No trailing period
- Keep under 72 characters

## Key Files to Check

- Commit messages (PR title and individual commit titles)
- [CONTRIBUTING.md](../../CONTRIBUTING.md) for the canonical reference

## Exclusions

- Merge commits generated by Git/GitHub
- Dependabot or automated dependency update commits
